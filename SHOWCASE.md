## 이 프로젝트가 증명하는 역량 (평가자용)

| 역량 | 어디서 |
|---|---|
| **Conformal Prediction** (분포가정 없는 유한표본 보장) | 검출 오탐율 ≤α 보장, 예측 오경보율 ≤1/c 보장 |
| **Eval 방법론** (faithfulness·대조군·부트스트랩 CI·유의성) | 레이어3 — 엔진 有/無 대조로 정직성을 정량화 |
| **인과추론** (DirectLiNGAM + 도메인 큐레이션) | 증상 vs lever 구분, robust 부분만 제품화 |
| **정직한 ML** (한계를 측정·negative result 보존·일반화 검증) | 모든 칸의 천장을 숫자로 명시, RG3 cross-line 검증 |

→ 핵심은 **검출 성능이 아니라 "측정된 신뢰"**. 검출은 commodity(논문 F1 97%)임을 알고, 노력을 eval·정직성에 투입.

---

## ★ 헤드라인 — moat는 레이어3 (정직성 측정)

**같은 LLM(gemma3:4b)을 엔진 有(우리) vs 無(vanilla)로만 가른 대조 실험:**

| 지표 | 우리(엔진) | vanilla | 결론 |
|---|---|---|---|
| 정상 40건 **confidently-wrong** | **7.5%** [2.6, 19.9] | **67.5%** [52, 80] | 차이 60%p, 95%CI [43, 77] → **통계적 유의** |
| 진짜결함 19 모드 | 정답9 / **기권8** / 오답2 | 정답2 / 기권0 / **오답17** | vanilla는 17/19 오진 |
| 정상범위 날조 | **0%** (실측 인용) | 75% | grounding-by-construction |

→ vanilla는 정상 제품의 2/3를 "불량"이라 **자신만만하게** 단정한다. 우리는 **모르면 기권**한다. 그 차이를 숫자로 증명.

## 가치 사다리 — 4칸, 각 칸의 성과 + 정직한 한계

```
서술 → 진단 ✅ → 처방 ✅ → 인과 ✅(robust) → 예측 ✅ → (자율)
```

| 칸 | 성과 | 정직한 한계 (측정함) |
|---|---|---|
| **진단** | 모드적중 82%, 검출 recall **58→89%**(지도 게이트), 오탐율 ≤5% **보장** | z<1σ 결함 불검출 = 데이터 한계 |
| **처방** | lever 직접조정 + 방향권고 | 상관 recourse 한계 → "공정검토"(자동수리 사칭 X) |
| **인과** | 증상(금형온도) 식별 → 상류 점검 안내 | 관측 단일라인 인과탐색 천장, 아티팩트 도메인 override |
| **예측** | 드리프트 lead-time 74~146 shot, 오경보율 **≤1% 보장** | 약한(1σ) 드리프트 천장 |

**통일 척추 = measured trust:** 검출 오탐율 ≤α → 예측 오경보율 ≤1/c. *같은 종류의 유한표본 보장이 사다리 두 칸에서 반복.*

---

## 정직함을 기능으로 — "측정해서 떨군 것"도 자산

- **검출 강화(58→89%)**: conformal 보장이 검출기 무관임을 이용해 score를 지도 GBM 불량확률로 교체.
  보장 유지(오탐율 4.6%≤5%). 못 잡은 2/19는 데이터에 흔적 없는 결함(모델 한계 아님)으로 분리 보고.
- **하이브리드 안전망**: 점수 블렌딩은 역효과(측정 후 폐기), 게이트 합집합은 known recall 보존하나
  novel coverage는 **공짜가 아닌 다이얼**임을 실증.
- **RG3 cross-line 검증**: 보장은 라인 간 전이 ✅(오탐율 4.9%≤5%), 검출력은 라인 의존 ❌(12.5%),
  시그니처는 부품 특수적 ❌ → **"단일공정 depth, 범용 copilot 아님" 전략을 실데이터로 정당화.**
- **ECOD vs Mahalanobis**: 동률(negative result) → Mahalanobis 유지. *추천 기법도 검증 후 채택.*

→ 모든 한계가 **측정된 trade-off**. 자율을 함부로 약속하지 않는 것 자체가 셀링포인트.
